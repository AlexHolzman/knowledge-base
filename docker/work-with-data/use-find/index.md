## Поиск образов

Когда дело доходит до управления Docker, поиск нужных образов является одной из ключевых задач. Давайте разберем, как это можно реализовать.

### Команда `docker images`

Первое, что стоит рассмотреть, это команда `docker images`. Она позволяет вам получить список всех образов, которые у вас есть в местном репозитории.

```bash
docker images
```

Запустив эту команду, вы увидите список всех доступных образов с их именами, тегами и размерами. Это базовая и крайне важная команда, которая позволяет держать под контролем локально сохраненные образы.

### Поиск образов в Docker Hub

Иногда нам нужно найти образы, которых нет на локальной машине. В этом случае высможете использовать Docker Hub — официальный репозиторий Docker, содержащий тысячи различных образов.

```bash
docker search <image_name>
```

Замените `<image_name>` на желаемое имя, и Docker вернет список соответствующих образов, доступных в Docker Hub. Это чрезвычайно полезно, когда вы ищете базовые образы для вашего проекта или популярные пакеты.

### Фильтрация выводимых данных

Возможно, показ всех образов не всегда удобен и нужен. В таких случаях фильтрация поможет вам сократить объем информации. Посмотрим на пример.

```bash
docker images --filter "dangling=true"
```

Этот запрос покажет только те образы, которые не связаны с каким-либо тегом и могут быть безопасно удалены для освобождения пространства.

## Поиск контейнеров

Теперь, когда мы знаем, как искать образы, давайте перейдем к поиску контейнеров. Ведь их управление и обнаружение также важны.

### Команда `docker ps`

Самый простой способ получить информацию о работающих контейнерах — это команда `docker ps`.

```bash
docker ps
```

Эта команда покажет все активные контейнеры на машине. Однако, если вам нужно увидеть все контейнеры — как работающие, так и остановленные — добавьте флаг `-a`.

```bash
docker ps -a
```

Как видите, теперь вы можете наблюдать за всеми вашими контейнерами, а не только за текущими.

### Фильтрация контейнеров

Как и в случае с образами, Docker позволяет фильтровать список контейнеров по разным характеристикам. Давайте рассмотрим пример.

```bash
docker ps --filter "status=exited"
```

Запуск этой команды вернет все контейнеры, которые были завершены или остановлены. Это полезно, если вы хотите очистить ненужные контейнеры или выяснить, какие конкретные контейнеры прекратили свою работу.

### Поиск при помощи `grep`

Помимо встроенной фильтрации, вы можете использовать `grep` для поиска контейнеров или образов, соответствующих определенному шаблону. Давайте посмотрим, как это делается.

```bash
docker ps -a | grep <part_of_name_or_id>
```

Замените `<part_of_name_or_id>` на часть ID или имени контейнера, и `grep` покажет все соответствия. Это быстрый способ отфильтровать нужные объекты по частичному совпадению.
