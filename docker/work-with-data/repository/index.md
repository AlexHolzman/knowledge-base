## Введение

Docker предоставляет мощную и гибкую платформу для работы с контейнерами, и одним из ее ключевых компонентов являются репозитории. Репозитории предоставляют разработчикам возможность создавать, хранить и публиковать образы контейнеров, делая их доступными для других пользователей. В этой статье мы рассмотрим, как именно можно работать с репозиториями в Docker и какие инструменты для этого предоставляет экосистема Docker.

## Работа с образами Docker

### Создание образа

Перед тем как начать работу с репозиториями, важно разобраться с созданием образов в Docker. Образ в Docker — это некий шаблон, который содержит все, что необходимо для запуска контейнера: код приложения, среду исполнения, библиотеки и зависимости.

Образы создаются на основе **Dockerfile** — текстового файла, содержащего пошаговые инструкции по сборке. Давайте рассмотрим пример:

```dockerfile
# Базовый образ с ОС Ubuntu
FROM ubuntu:20.04 

# Устанавливаем обновления и необходимые пакеты
RUN apt-get update && apt-get install -y nginx 

# Копируем внутренние файлы на образ
COPY . /usr/share/nginx/html 

# Открываем порт 80 для доступа
EXPOSE 80 

# Указываем команду для запуска Nginx при старте контейнера
CMD ["nginx", "-g", "daemon off;"]
```

Обратите внимание, как мы последовательно создаем среду для нашего приложения.

### Сборка образа

После того как Dockerfile создан, следующим шагом является сборка образа. Команда `docker build` используется для этого:

```sh
docker build -t my-nginx-image .
```

- `-t my-nginx-image` — задает имя нашему образу.
- `.` — указывает, что Dockerfile находится в текущей директории.

### Загрузка образа в репозиторий

Теперь, когда образ собран, настало время его публикации в репозитории. Docker Hub — это наиболее популярный публичный репозиторий для хранения и распространения Docker образов. Чтобы загрузить ваш образ на Docker Hub, выполните следующие шаги:

1. **Войдите в Docker Hub**

   ```sh
   docker login
   ```

   Введите ваши учетные данные, чтобы аутентифицироваться.

2. **Добавление метки вашего образа для docker push**

   ```sh
   docker tag my-nginx-image yourusername/my-nginx-image
   ```

3. **Выгрузка образа на Docker Hub**

   ```sh
   docker push yourusername/my-nginx-image
   ```

Теперь ваш образ доступен для других пользователей через репозиторий Docker Hub.

## Управление репозиториями через CLI

Docker CLI — мощный инструмент для управления образами и контейнерами. Давайте рассмотрим некоторые распространенные команды, используемые при работе с репозиториями.

### Просмотр доступных образов

Вы можете получить список всех локально доступных образов с помощью:

```sh
docker images
```

Эта команда выдаст имена, теги и идентификаторы всех образов, которые вы скачали или создали на вашей машине.

### Удаление образов

Если вам нужно очистить пространство или удалить ненужный образ, используйте следующую команду:

```sh
docker rmi имя_образа
```

Замените `имя_образа` на имя или идентификатор образа, который вы хотите удалить.

### Поиск образа в Docker Hub

Чтобы найти конкретный образ в Docker Hub, вы можете использовать:

```sh
docker search nginx
```

Как видите, это покажет вам список всех образов, доступных в публичных репозиториях, связанных с определенным запросом, в данном случае `nginx`.
